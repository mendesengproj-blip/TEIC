# M2 — Lorentz invariance of the mass proxy (conecta R1)

**Verdict: B** — the mass *proxy* is Lorentz invariant, but the result is inherited
from R1, not an independent particle mass.

## Question
M1 found no measurable inertial rest mass, so the only mass-like scalar the network
offers is the **content of a causal diamond** (the connectivity/action scale T15/T16
reached for). A genuine mass must be Lorentz invariant. We test it directly.

## Method (the clean isotropy test, as in R1 Panel B)
Fix an **invariant** proper time τ₀=6. Build the causal diamond
`A=(0,0) → B=(τ₀ cosh φ, τ₀ sinh φ)`; the identity `cosh²−sinh²=1` keeps the
invariant interval = τ₀ for every rapidity φ (pure geometry, no dilation formula).
The mass proxy = number of events `N` inside the diamond. A Lorentz-invariant
quantity must give a **constant** N (up to Poisson noise) as φ varies. Poisson
network (ρ=8, 20 seeds, 12 rapidities up to β≈0.99) vs the regular lattice control.

## Results

| | mean count | CV across boosts |
|---|---|---|
| expected `ρτ₀²/2` | 144.0 | — |
| **Poisson** | 144.4 | **1.5 %** (invariant) |
| **lattice** | 150.8 | **14.1 %** (direction-dependent) |

## What it means
The diamond content is **Lorentz invariant for the Poisson network** (CV 1.5 %) and
**frame-dependent for the lattice** (CV 14.1 %) — a clean ~10× separation. This is a
real result, but it is **R1's Poisson Lorentz-invariance applied to the connectivity
scale**, not an independently derived particle mass. Combined with M1: the network
*does* carry a Lorentz-invariant connectivity scalar, but it behaves like an
action/volume content, not like an inertial rest mass that resists acceleration.
